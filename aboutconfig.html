Justin.tv Stream Notifications

This add-on displays a notification when a channel on justin.tv resp. twitch.tv goes live. Which channels are checked for being live is fully customizable.

about:config Settings

Justin.tv Stream Notifications has some advanced settings, only customizeable via about:config.

Open about:config

To open about:confg, open a new tab and enter "about:config" into the URL-bar (without the quotes). If you never used about:config before you will see a message, asking you to be cautious about what you do. You can break Firefox with about:config tweaks pretty easily.

Available Settings

Justin.tv Stream Notifications offers the following hidden settings:

  • queue.ratio (Default: 2) Defines the part of the queue fetched minimally at a time. It will always fetch at least one request. the value is reciproke to the actual used ratio.
  • queue.maxRequestBatchSize (Default: 8) The maximal number of requests fetched at once.
  • panel.minWidth (Default: 212) The minimal width in pixels of the panel.
  • panel.advancedStyle (Default: true) When enabled uses colors from the channel for hover styling. To restore old behaviour, disable this preference.
  • justintv.backgroundImageName (Default: channel_background_image_url) The property name from the fetched channel object for the image displayed in the background of the hover effect. Alternatives: screen_cap_url_large (screenshot of the stream)
  • justintv.apiKey API key for justin.tv for higher rate limit.
  • justintv.ignoreRatelimit (Default: false) Completely ignores the rate-limit check
  • twitch.clientID Twitch developer application client ID.
  • twitch.backgroundImageName (Default: background) The property from the fetched channel object which is used as background on hover in the panel.
  • debug (Default: false) Disables SS.
